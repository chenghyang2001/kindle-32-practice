加碼 · Claude Code vs Claude Cowork 任務分流

你已經在用 Claude Code CLI——這張速查表幫你判斷哪些事該改丟給 Cowork。取材自全書 13 章範例。

① 速查表：這件事該用誰？

留在 Claude Code（CLI）

  • 寫 / 改功能、修 bug、重構
  • 跑測試、看 stack trace
  • git 操作、開分支、管 worktree
  • 直接讀寫 repo、執行 shell 指令
  • 需要「動到程式碼本身」的一切

改用 Claude Cowork

  • 整理本機檔案 / 照片 / 下載夾
  • 接 Gmail / Calendar / iMessage 辦事
  • 上網研究 → 產報告 / 摘要 / 筆記
  • 產 PPT / Excel / Word / 海報
  • 資料清理 → 分析 → 視覺化
  • 每天 / 每週自動跑的例行事（排程）
  • Code Review checklist、架構文件、建 MCP
三個 Cowork 才有、CLI 給不了的決定性差異

1. 排程（Scheduled）：設「每天 8:00 自動彙整昨天新聞」——CLI 不會自己醒來跑。

2. Connector 直連：Gmail / Calendar / Drive / iMessage 直接讀寫，不必自己串 API。

3. 文件 Skill：直接吐出 .pptx / .xlsx / .docx / 海報這類非程式產出。

口訣：寫程式碼本身用 Code；圍繞程式碼的文件、審查、工具搭建，與一切非程式雜活用 Cowork。

③ 現在就試：把一件雜活丟給 Cowork

挑一件你平常會在終端機外「手動做」的雜活，改用 Cowork 委派一次，體會差別。例如：

📅 幫我看這週 Google Calendar，把有衝突的會議列出來，並建議怎麼調整。（接 Calendar Connector）
📊 讀這份 sales.csv，清理後產生一份含月趨勢圖與前十大客戶的互動報表。（資料分析 Skill）
📄 從這個 repo 的原始碼，產生一份架構設計文件（含模組關係與資料流）。（圍繞程式碼的文件——Cowork 比 CLI 更適合交付給人看的產物）

做完你會發現：這些「不是寫程式、但要跑好幾步」的事，Cowork 一句委派就自己完成，正好補上 Claude Code 不碰的那一塊。
