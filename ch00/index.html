① 先回顧：AI 能力演進的四個階段

Chatbot

聊天機器人
一問一答

你問一句、AI 答一句。不存取檔案、不主動做事，代表產品是 ChatGPT、Claude Chat、Gemini。

Coding Agent

程式開發代理
2023 年起

能讀寫程式檔案、執行終端機指令、管理 Git，但世界僅限於「程式碼和終端機」。代表：GitHub Copilot、Cursor、Claude Code。

Computer Agent

電腦操控代理
2024 年底

靠「看螢幕截圖 → 決定點哪裡」操作任何 GUI 應用程式，速度慢、容易出錯。代表：Claude Computer Use、Operator。

Desktop Agent

桌面代理
2025 年，Claude Cowork

整合多種互動方式，自主規劃、拆解步驟、存取檔案、排程、跨雲端服務，真的動手完成任務。

一句話：Chatbot 只動嘴、Coding Agent 只碰程式碼、Computer Agent 靠截圖操作、Desktop Agent 真的動手完成多步驟任務。本書聚焦 Claude Cowork（Desktop Agent）。

③ 接回真實操作：到 Cowork 做一次 demo

光做題不夠，打開 Claude Desktop → 切到 Cowork 分頁，把書中第一個示範任務貼進去實際跑一次，親眼見證 Chatbot 和 Desktop Agent 的根本性差異：

1️⃣ 在任務輸入區貼上：
請幫我在桌面建立一個「Cowork測試」資料夾，裡面放三個檔案：todo.txt（內容是今天的待辦事項範例）、notes.md（內容是會議記錄範例）、expense.csv（內容是本月費用範例，含標題列和五筆資料）。
2️⃣ 按 Let's go →，觀察右側的 Progress 面板：Claude 會自己拆解成「建立資料夾 → 建立 todo.txt → 建立 notes.md → 建立 expense.csv」四個步驟，並在需要動手前先跳出權限請求（Allow for this task / Allow once / Deny）讓你確認。
3️⃣ 全部完成後（Progress 顯示 4 of 4），回到桌面確認真的多了一個「Cowork測試」資料夾、裡面真的有三個內容合理的檔案——這就是「只動嘴」和「真的動手」的差別。

⚠️ Cowork 目前是研究預覽版（research preview），每一步涉及檔案操作都會先請你確認權限，放心它不會偷偷動手。
