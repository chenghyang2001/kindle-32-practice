① 先回顧：三種模式

Chat · 對話

「一起思考」
What shall we think through?

你問一句、AI 答一句，結束就結束。適合問答、寫作、翻譯、腦力激盪。

Cowork · 委派

「幫你完成待辦」
Let's knock something off your list

你描述「想要的結果」，AI 自主規劃、拆解子任務、存取檔案、上網，交回成果。可排程、可多裝置。

Code · 開發

開發者的終端機
整合 Claude Code

寫程式、執行指令、管理 Git、連 GitHub repo 與分支。非開發者可先跳過。

比較項目ChatCoworkCode
核心用途一問一答的對話指派多步驟任務程式開發

存取檔案需手動上傳直接存取本機檔案直接存取專案目錄
執行時間即時回覆可能需要數分鐘依任務而定
排程功能無有（可設定自動執行）無
多裝置無有（Dispatch）無
適合場景問答、寫作、翻譯檔案整理、報表產生、研究分析寫程式、管理 Git

一句話：Chat 是對話、Cowork 是委派、Code 是開發。本書聚焦 Cowork。

③ 接回真實操作：到 Cowork 做一次 demo

光做題不夠，打開 Claude Desktop → 切到 Cowork 分頁，把書中最經典的第一個委派任務貼進去實際跑一次：

1️⃣ 在任務輸入區貼上：
幫我把「下載」資料夾裡的檔案，按類型整理成子資料夾（圖片、PDF、影片、文件各一個）。
2️⃣ 按 Let's go →，然後去泡杯咖啡——Cowork 會自己規劃步驟、存取檔案、分類，不用你盯著。
3️⃣ 回來看它交回的成果，體會「委派」和「聊天」的根本差別：你只描述了想要的結果，其餘它自己完成。

⚠️ Cowork 目前是研究預覽版（research preview），第一次跑會請你確認檔案存取權限，放心它每步都會問過你。
