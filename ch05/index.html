① 先回顧：三個 Connector

Google Calendar

Web Connector · Google OAuth

建立／修改／刪除行事曆事件，重複事件用 RRULE 語法，可在同一對話中用自然語言追加修改。需要「寫入」權限（Write/delete tools）。

Gmail

Web Connector · Google OAuth

搜尋、讀取、分類郵件，產生摘要報告與待辦事項清單。書中範例只需「讀取」權限，不會幫你發信或回信。

iMessages

Desktop Connector · 本機存取

直接讀取／發送 Mac 上訊息 App 的對話。需要「完整磁碟取用權限」，沒開也會自動切換用 computer use 觀察畫面。

一句話：Calendar／Gmail 是 Web Connector（走 Google OAuth），iMessages 是唯一的 Desktop Connector（直接操作本機 App）。Gmail 和 iMessages 因為任務需要產生文件，都自動載入了 docx Skill。

③ 接回真實操作：到 Cowork 做一次 demo

光做題不夠，打開 Claude Desktop → 切到 Cowork 分頁，先確認 Customize → Connectors 裡 Google Calendar 已連接且開了「寫入」權限（見書中 5-1-1），再把本章範例 4 的任務貼進去實際跑一次：

1️⃣ 在任務輸入區貼上：
請幫我規劃下週（4/14 到 4/20）的一週行程，直接在我的 Google Calendar
上建立事件。我是一位自由工作者，需要安排以下事項：
1. 每天早上 9:00-9:30 晨間規劃（週一到週五重複）
2. 週一 10:00-12:00 客戶會議（線上，Google Meet）
3. 週一 14:00-16:00 專案開發時間
4. 週二 10:00-11:30 和設計師討論 UI 修改
5. 週二 13:00-14:00 午餐約會（信義區餐廳）
每個事件請加上適當的描述和地點。
2️⃣ 按 Let's go →，第一次授權時選 Allow for this task，觀察右側 Progress 面板逐步打勾，以及 Context 面板載入的 Google Calendar Connector。
3️⃣ 完成後打開 calendar.google.com 週檢視驗證：晨間規劃應該是「一個」重複事件（RRULE），不是五個獨立事件——這正是本章教的「智慧處理重複事件」。接著再用同一對話追加一句自然語言修改指令，體會 Claude 記得所有已建立事件、直接用 Update API 修改的差別。

⚠️ 若少勾了 OAuth 頁面第三項「查看、編輯、共用及永久刪除」權限，Claude 會只能讀取無法建立事件，需回 Connectors 頁面 Disconnect 後重新連接。
