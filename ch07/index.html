① 先回顧：三個辦公文件 Skill

pptx · 簡報

「筆記轉簡報」
指令驅動

把 Markdown 筆記轉成 PowerPoint。Claude 會依內容性質選版面（數據用卡片、時程用時間軸、預算用圖表），並自動做視覺 QA。

xlsx · 試算表

「從零建表」
指令驅動

用 openpyxl 建立含公式、條件格式、多工作表的 Excel。完成後用獨立 recalc.py 腳本重新計算所有公式，零錯誤才交付。

doc-coauthoring · 協作撰稿

「協作撰寫正式文件」
協作驅動

不會拿到指令就埋頭寫，而是先問你關鍵問題了解背景，分三階段（脈絡蒐集→逐節打磨→讀者測試）完成，最後用子代理做讀者測試。

③ 接回真實操作：到 Cowork 做一次 demo

光做題不夠，打開 Claude Desktop → 切到 Cowork 分頁，把書中範例 10 的任務貼進去實際跑一次（先在桌面準備一份 Markdown 會議筆記，例如「會議筆記-AI策略規劃.md」）：

1️⃣ 在任務輸入區貼上：
我的桌面上有一份會議筆記檔案（會議筆記-AI策略規劃.md），是上週 AI 策略規劃會議的紀錄。請幫我：1) 讀取這份筆記的完整內容 2) 將筆記內容轉換成一份專業的 PowerPoint 簡報（.pptx），包含：封面頁、目錄頁、每個主題一張投影片 3) 每張投影片要有清楚的標題和重點條列 4) 加入適當的表格來呈現數據 5) 最後一張是 Q&A 頁面。請產生約 10-12 張投影片的簡報。
2️⃣ 按 Let's go →，過程中若出現「Claude wants to use start_process from Desktop Commander」的權限請求，確認指令只是 cat 讀檔後按 Allow for this task。
3️⃣ 觀察 Progress 面板的六步驟：讀取筆記 → 讀 pptx 技能文件 → 撰寫簡報程式碼 → 產生 PPTX → 視覺品質檢查（QA）→ 修正問題並交付。留意 Claude 有沒有自己抓出投影片截斷之類的問題並自動修正。

⚠️ Cowork 目前是研究預覽版（research preview），每個檔案存取 / 執行指令的請求都會先問過你，放心確認即可。
