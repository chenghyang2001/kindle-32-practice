③ 接回真實操作：到 Cowork 做一次 demo

光做題不夠，打開 Claude Desktop → 切到 Cowork 分頁，先在 Settings > Customize > Connectors 確認已連接 GitHub，再把書中範例 19 的審查任務貼進去實際跑一次（repo 和 PR 編號換成你自己的）：

1️⃣ 在任務輸入區貼上：
請幫我審查 GitHub repo joshhu/cowork-demo-api 的 Pull Request#1。這個 PR 新增了待辦事項 API 的進階功能（優先級、標籤、搜尋、統計端點）。請逐一檢查：1) 程式碼品質與可讀性 2)SQL injection 等安全性問題 3) 錯誤處理是否完善 4)API 設計是否符合 RESTful 慣例。最後產生一份 Code Review Checklist 的 Word 文件。
2️⃣ 按 Let's go →，觀察右側 Progress 面板依序完成七個步驟，Context 面板會顯示已載入 docx Skill。
3️⃣ 完成後點開產出的 code-review-pr1.docx，體會「不只挑問題、還附上可直接複製的修正程式碼」的差異——這正是委派給 Cowork 和自己肉眼看 diff 的根本不同。

⚠️ Cowork 目前是研究預覽版（research preview），第一次跑會請你確認 GitHub 存取權限，放心它每步都會問過你。
